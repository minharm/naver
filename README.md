프로젝트 폴더 구조 예시:

```text

    upload_package/
```

주의: `projects/`는 개인 작업 데이터이므로 배포 ZIP에는 포함하지 않습니다.

## v0.5.6 수정 사항

### safe context 우회 방지

이전에는 문장에 `사진상으로는`, `검색 자료 기준으로는`, `공개 자료 기준으로는` 같은 표현이 있으면 후처리를 건너뛰었습니다.

v0.5.6에서는 safe context가 있어도 아래와 같은 고위험 표현은 계속 완화합니다.

- 사진상으로는 정말 맛있었어요
- 검색 자료 기준으로는 가성비가 정말 좋았어요
- 공개 자료 기준으로는 사장님이 친절했어요
- 검색 자료 기준으로는 재방문 의사가 있어 보여요

### 검색 결과 신뢰등급 분리

검색 결과를 단순 유효/무효로만 보지 않고 아래 등급으로 나눕니다.

- high: 70점 이상, 본문 사실 근거 사용 가능
- medium: 45~69점, 분위기/후기 소재 참고
- low: 25~44점, 보관은 하되 사실 단정 근거 사용 금지
- excluded: 25점 미만

`use_for_facts=True`는 high 등급에만 붙습니다.

### 업체 정보 분석 프롬프트 강화

`verified_facts`는 `use_for_facts=True`인 고신뢰 출처에서 확인된 내용만 넣도록 프롬프트를 강화했습니다.

중신뢰/저신뢰 결과는 분위기 참고, 후기 소재 참고만 가능하며 본문 사실 단정 근거로 쓰지 않습니다.

### 업종별 메뉴/서비스 키워드 분리

structured_facts의 메뉴/서비스 추출을 업종별로 나누었습니다.

- 중식: 짜장, 짬뽕, 탕수육, 볶음밥 등
- 카페: 아메리카노, 라떼, 디저트, 케이크 등
- 고깃집: 삼겹살, 갈비, 한우, 냉면 등
- 키즈카페: 입장료, 보호자, 놀이시설, 파티룸 등
- 병원: 진료과목, 예약, 주차, 진료시간 등
- 공방: 원데이클래스, 체험, 수업, 재료비 등
- 숙박: 객실, 체크인, 체크아웃, 조식 등

실행:

```powershell
pytest
```

또는 더 확실하게:

```powershell
python -m pytest
```

- `다녀왔다`, `먹어봤다`, `주문했다`, `아이들이 잘 먹었다` 같은 표현 제한
- 사진만 보고 맛, 친절도, 가성비, 주차 편의, 대표 메뉴를 단정하지 않음
- verified_facts에 없는 전화번호/영업시간/주차/예약/배달 정보는 단정하지 않음
- 확인 필요 정보는 방문 전 확인 안내 수준으로만 작성

STEP 4에 `직접 경험 메모/방문 메모` 입력칸을 추가했습니다.

## 4. 프로젝트 관리 기능 추가

사이드바에 프로젝트 관리 기능을 추가했습니다.

- 현재 작업 프로젝트로 저장
- 최근 프로젝트 목록
- 프로젝트 불러오기
- 프로젝트 삭제
- 프로젝트 복제
- 프로젝트 이름 변경

프로젝트는 `projects/프로젝트명/project.json` 구조로 저장됩니다.

주의: `projects/`는 개인 작업 데이터이므로 배포 ZIP에는 포함하지 않습니다.

## 6. 경로 저장 방식 개선

업로드 이미지/영상, 가공 이미지 경로는 가능한 경우 프로젝트 기준 상대경로로 저장합니다.

```text
uploads/images/sample.jpg
output/processed_images/sample_processed.png
```

절대경로가 저장되어 다른 PC에서 깨지는 문제를 줄였습니다.
